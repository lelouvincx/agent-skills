## Environment selection

Resolution uses `AGENTS_REGISTRY_ENV` when set. If it is not set, `project-resolve` tries to auto-detect the environment from `projects.yaml`.
Set `AGENTS_REGISTRY_WORKSPACE_ROOT` to override the selected environment's default workspace root when a host uses a different checkout path.

## Usage

```bash
# prerequisite: uv and uvx on PATH; pyyaml/rapidfuzz are provided by uvx
command -v uv >/dev/null && command -v uvx >/dev/null

project-resolve logseq --path
project-resolve "log this to logseq" --github
project-resolve dbt --json
project-resolve --list
AGENTS_REGISTRY_ENV=vps project-resolve prefect --path
```

## Matching policy

`project-resolve` resolves in this order:

1. Exact project key.
2. Exact alias.
3. Exact GitHub repository basename, when unique.
4. Deterministic fuzzy match through `rapidfuzz` across keys, aliases, repo slugs, repo basenames, and descriptions.

If fuzzy matching is not clearly unique, the resolver fails and prints candidates instead of guessing.
