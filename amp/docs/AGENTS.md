# Amp capability documentation instructions

- Load and apply the `govuk-style` skill when writing or editing capability documents in `tools/*.md` or RFCs in `rfcs/rfc-*.md`. This keeps the documents plain, active, front-loaded, and easy to review line by line without losing technical detail.
- Treat `tools/*.md` as the source of truth for Amp plugin capabilities.
- Update the relevant capability document before changing a corresponding plugin, skill, tool, or agent instruction.
- Keep downstream artifacts aligned with the documented contract.
- Treat `issues/*.md` as durable rationale records. Preserve their original intent and evidence while updating resolution status under `issues/_schema.md`.
